    Essential API Calls

    Below are the essential API calls which allow interaction with Arcube’s AI Recommendation Engine. For each integration method, we indicate which API calls are required.

    Bearer Token

    All calls require a valid Bearer token obtained from the authentication endpoint. The required credentials are client_id and client_secret, which will be provided by Arcube. See: Authentication.

    6.1 Ingest Passenger Data

    Description

    Insert trip and passenger context into Arcube to enable personalization.

    Request URL Parameter

    Replace ${arcube-url} with your assigned Arcube environment URL (e.g., https://api.arcube.com).

    HTTP Method: POST

    URL: ${arcube-url}/v1/journey-api/passenger-data

    Request Headers

    • Authorization: Bearer <access_token>
    • Content-Type: application/json
    • Idempotency-Key: <uuid> (recommended)

    HTTP Response

    202 Accepted — request accepted for asynchronous processing.

    Notes

    • Include the lead passenger’s email if you want Arcube to send transactional/marketing emails.
    • Provide either pnr or sessionId (or both) consistently across subsequent calls.

    6.2 Serve Recommendations

    Description

    Retrieve recommendations for a specific PNR or session and point of sale.

    Request URL Parameter

    Replace ${arcube-url} with your assigned Arcube environment URL (e.g., https://api.arcube.com).

    HTTP Method: POST

    URL: ${arcube-url}/v1/aggregation-api/servePassengerRecommendations

    Request Headers

    • Authorization: Bearer <access_token>
    • Content-Type: application/json

    HTTP Response

    200 OK

    Request Payload Sample

    {
  "tenant": "6864f9f94eab200cd4765cc0",
  "sessionId": "73-ara43",
  "pointOfSale": "booking-flow"
}

    Request Payload JSON Schema

    {
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "required": ["pointOfSale", "tenant"],
  "properties": {
    "pnr": { "type": "string" },
    "sessionId": { "type": "string" },
    "pointOfSale": { "type": "string" },
    "tenant": { "type": "string" }
  },
  "anyOf": [
    { "required": ["pnr"] },
    { "required": ["sessionId"] }
  ],
  "additionalProperties": false
}

    6.3 Confirm Order

    Description

    Confirm an order after successful payment. Arcube will place orders with relevant vendors and record the transaction.

    Request URL Parameter

    Replace ${arcube-url} with your assigned Arcube environment URL (e.g., https://api.arcube.com).

    HTTP Method: POST

    URL: ${arcube-url}/v1/order-api/confirmOrder

    Request Headers

    • Authorization: Bearer <access_token>
    • Content-Type: application/json
    • Idempotency-Key: <uuid> (required for safe retries)

    HTTP Response

    200 OK (full success) — or 207 Multi-Status / 200 with status="PARTIAL-SUCCESS" for partial fulfillment.

    Sample Request Payload

    {
  "sessionId": "fkdsjf-f123-4567",
  "selectedAncillaries": [
    {
      "vendor": "brb",

      "price": { "amount": 35.00, "currency": "USD" }
    },
    {
      "vendor": "mozio",
      "id": "e9b1d12f-8f1d-4f32-9c3e-8d928c1b3bb5",
      "price": { "amount": 140.00, "currency": "USD" },
      "metadata": {
        "result_id": "e9b1d12f-8f1d-4f32-9c3e-8d928c1b3gg5",
